# Base Skills — Quick Reference

| # | Skill | One-line summary | Self-check question |
|---|-------|-----------------|---------------------|
| 1 | [pareto-focus](skills/pareto-focus/SKILL.md) | Identify and protect the 20% of effort that delivers 80% of value | What is the 80th percentile outcome and which work items serve it? |
| 2 | [hard-choice](skills/hard-choice/SKILL.md) | Evaluate easy-path (today) vs hard-path (2 years) before committing | Have I written down both paths and produced a Decision Record? |
| 3 | [solid-gate](skills/solid-gate/SKILL.md) | Check every class/module/function against all five SOLID principles | Which SOLID principle is violated, and at what severity? |
| 4 | [dry-kiss-yagni](skills/dry-kiss-yagni/SKILL.md) | No abstraction without 3 consumers, no speculative code, no deep nesting | Does this abstraction serve 3+ current consumers? |
| 5 | [verification-evidence](skills/verification-evidence/SKILL.md) | Never claim completion without observable evidence (IDENTIFY→RUN→READ→VERIFY→CLAIM) | What observable evidence confirms this works? |
| 6 | [design-challenge](skills/design-challenge/SKILL.md) | Adversarially evaluate every technical proposal; steelman alternatives before rejecting | Have I steelmanned every rejected alternative? |
| 7 | [future-proof](skills/future-proof/SKILL.md) | Make the 2-year maintenance cost visible before committing | What is the change-cost (files touched) for a single requirement change? |
| 8 | [scout](skills/scout/SKILL.md) | Leave every touched file better than you found it; produce an Improvement Record | Have I produced an Improvement Record for every file I touched? |
| 9 | [anti-slop](skills/anti-slop/SKILL.md) | Flag AI-generated boilerplate, empty handlers, and single-consumer abstractions | What breaks if I delete this code right now? |
| 10 | [review-gate](skills/review-gate/SKILL.md) | Classify every finding by severity (CRITICAL/HIGH/MEDIUM/LOW) and block accordingly | Are all CRITICAL findings resolved and all HIGH findings documented? |
| 11 | [root-cause](skills/root-cause/SKILL.md) | Apply 5 Whys before any fix; distinguish symptom from root cause | Have I documented the root cause chain (minimum 3 Whys)? |
| 12 | [scope-discipline](skills/scope-discipline/SKILL.md) | Define IS / IS NOT scope before starting; never fix outside it | Is this change in my IS scope list? |
| 13 | [architecture-evidence](skills/architecture-evidence/SKILL.md) | Every significant decision needs an ADR with 2+ alternatives evaluated | Does this have a written ADR with the rejected option's strongest argument? |
| 14 | [naming](skills/naming/SKILL.md) | All identifiers must pass the explains-itself test; ban generic names | Does every new identifier pass the explains-itself test? |
| 15 | [approval-gate](skills/approval-gate/SKILL.md) | Confirm before >3 files, irreversible actions, architecture decisions, or ambiguity | Is the task unambiguous and low-risk — or have I reformulated and confirmed? |
